BEME

Bounty-driven Evolutionary Market Ensemble — multi-label classification under class imbalance, powered by prediction markets and evolutionary selection.

v0.2.0 Python ≥ 3.8 scikit-learn compatible Multi-label

📦 Installation

BEME is not yet on PyPI. Install it directly from the GitHub repository:

  1. Clone the repository 
    git clone https://github.com/mesely/beme.git
cd beme
  2. Install in editable mode 
    pip install -e .

    The -e flag installs the package in editable mode — Python points directly at the cloned folder, so any changes you make to the source are immediately reflected without re-installing.

  3. Verify the installation 
    python -c "from beme import BemeMarket; print('BEME ready')"
No PyPI required. pip install -e . is the standard way to install a local package. After this single command, import beme works from anywhere in your Python environment — scripts, notebooks, other projects.

Google Colab

Colab automatically adds /content to sys.path, which causes Python to pick up the repo root as a namespace package instead of the real beme package. Skip pip install -e . and use this instead: 

!git clone https://github.com/mesely/beme.git /content/beme

⚡ Quick Start

v0.2.0 — One-liner

from beme import BemeMarket, MultiLabelWrapper
from sklearn.multioutput import MultiOutputClassifier
from sklearn.linear_model import LogisticRegression

model = BemeMarket(bounty_map='auto').fit(
    X_train, y_train,
    base_model=MultiLabelWrapper(MultiOutputClassifier(LogisticRegression(max_iter=1000)))
)
y_pred = model.predict(X_test)

Full example

from beme import BemeMarket, MultiLabelWrapper
from sklearn.linear_model import LogisticRegression
from sklearn.multioutput import MultiOutputClassifier
from sklearn.datasets import make_multilabel_classification
from sklearn.model_selection import train_test_split
from sklearn.metrics import f1_score

# 1. Prepare data
X, y = make_multilabel_classification(
    n_samples=1000, n_features=50, n_classes=10, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# 2. Wrap your sklearn estimator
base_model = MultiLabelWrapper(
    MultiOutputClassifier(LogisticRegression(max_iter=1000))
)

# 3. Fit — initialises market + runs all trading days in one call
market = BemeMarket(
    n_funds=50,
    leverage=25.0,
    temperature=1.5,
    bounty_map='auto',      # auto-compute bounties from class frequencies
    rebalance_period=100,
)
market.fit(X_train, y_train, base_model=base_model)

# 4. Predict on held-out test set (no balance updates)
y_pred = market.predict(X_test)
print(f1_score(y_test, y_pred, average='macro'))

# 5. Inspect portfolio
print(market.portfolio_summary())
Tip: Use market.portfolio_summary() to see which agents have accumulated the most wealth — high-balance agents are your best classifiers.

🏛 BemeMarket

The main class. Import it and everything is accessible through one object. 

from beme import BemeMarket

market = BemeMarket(
    n_funds=50,
    leverage=25.0,
    temperature=1.5,
    bounty_map=None,          # uses DEFAULT_BOUNTY_MAP
    rebalance_period=100,
    sector_map=None,          # reserved for future use
)

Constructor Parameters

Parameter Type Default Description
n_funds int 50 Number of hedge fund agents in the market. More funds = more diversity but slower training.
leverage float 25.0 Scales the profit/loss magnitude on every trading day. Higher values cause faster convergence but increase the risk of agents going bankrupt on noisy data.
temperature float 1.5 Controls how much the richest agent dominates the consensus. Low T (→ 0): winner-takes-all. High T (>> 1): equal voting. For NLP / sparse text data keep this between 1.5–2.5.
bounty_map dict | 'auto' | None None Per-class reward multipliers {class_index: float}.

'auto' — computed from class frequencies at fit time:
β_c = 1.0 + log(max_count / count_c)

None — uses default map {1:1.5, 2:2.0, 7:1.5, 8:2.0}
{} — disables bounty entirely (all classes equal)
rebalance_period int 100 Number of trading days between automatic evolutionary rebalancing cycles. Every H days, the weakest agent in each sector is replaced by a clone of the strongest.
sector_map dict | None None Reserved for future custom sector definitions. Currently unused.

🎯 fit v0.2.0

market.fit(X, y, base_model=None) → self

High-level scikit-learn compatible training entry point. Combines initialize_market() + the full training loop into one call. Returns self for method chaining.

If bounty_map='auto' was set at construction, bounties are computed from class frequencies in y before agents are spawned. Handles both dense arrays and scipy.sparse matrices transparently.

Parameters

ParameterTypeDescription
X array-like or sparse (n_samples, n_features) Feature matrix. Dense arrays and scipy sparse matrices are both supported.
y array-like or sparse (n_samples, n_classes) Binary label matrix. Sparse rows are automatically densified per sample.
base_model sklearn estimator or None Required on first call. Ignored if initialize_market() was already called. Wrap with MultiLabelWrapper for MultiOutputClassifier.

Returns

self — enables method chaining.

Progress Tracking

If tqdm is installed, a live progress bar is shown automatically. Otherwise a simple print statement reports the number of trading days. 

pip install tqdm   # optional but recommended for large datasets
# Method chaining
y_pred = (
    BemeMarket(n_funds=50, bounty_map='auto')
    .fit(X_train, y_train, base_model=base_model)
    .predict(X_test)
)

# Re-fit on new data without re-initialising agents
market.fit(X_new, y_new)   # base_model not needed — funds already exist
Do not use fit() on test data. It updates agent balances using the true labels. Use predict() for inference on held-out sets.

🚀 initialize_market

market.initialize_market(base_model, X_train, y_train) → None

Spawns n_funds agents. Each agent receives a stratified 70% bootstrap of the training data and independently trains a deep copy of base_model.

After this call, market.funds is populated with agent dicts containing their fitted models, starting balance (100.0), a random risk factor, sector assignment, and active class list.

Parameters

ParameterTypeDescription
base_model sklearn estimator Must implement fit(X, y) and predict_proba(X). It is deep-copied n_funds times, so passing an already-fitted model is fine — each agent starts from the same architecture but trains on a different data subset.

For standard sklearn multi-label pipelines, wrap with MultiLabelWrapper.
X_train array-like (n_samples, n_features) Feature matrix.
y_train array-like (n_samples, n_classes) Binary label matrix. Each row is a multi-hot vector.
Sector assignment: Agents 0 → n_funds/2-1 are placed in the COMMODITY sector (active on classes 1, 2, 4, 7, 9). The remaining agents go to FINANCE (classes 0, 3, 5, 6, 8). Each class is priced exclusively by its sector agents. 
# Example — with a MultiOutputClassifier wrapper
from beme import BemeMarket, MultiLabelWrapper
from sklearn.multioutput import MultiOutputClassifier
from sklearn.ensemble import RandomForestClassifier

market = BemeMarket()
base = MultiLabelWrapper(MultiOutputClassifier(RandomForestClassifier(n_estimators=50)))
market.initialize_market(base, X_train, y_train)

📈 run_trading_day

market.run_trading_day(x_i, y_true) → np.ndarray (n_classes,)

Processes one labelled sample. This is the core training step. Call it once per sample in your training loop.

Internally it: collects all agent predictions → computes market consensus prices → calculates each agent's P&L → updates balances → returns the binary prediction. Automatically triggers evolution every rebalance_period calls.

Parameters

ParameterTypeDescription
x_i array-like (1, n_features) Single sample. Must be 2-D with shape (1, n_features).
y_true array-like (n_classes,) Ground truth binary vector for this sample.

Returns

np.ndarray of shape (n_classes,) — binary predictions based on the market consensus before the balance update (so the returned prediction is what the market believed before seeing the true label).

for i in range(len(X_train)):
    y_hat = market.run_trading_day(
        X_train[i:i+1],   # shape (1, n_features)
        y_train[i]        # shape (n_classes,)
    )
Do not call this on test data. run_trading_day updates agent balances using the true label. Use predict() for inference on held-out sets.

🔮 predict

market.predict(X) → np.ndarray (n_samples, n_classes)

Batch inference. Does not update balances or trigger evolution. Safe to call on validation and test sets.

v0.2.0 optimisation: each agent's predict_proba is called once on the full X matrix (O(n_funds) calls) instead of once per sample (O(n_samples × n_funds) calls). On large test sets this is orders of magnitude faster.

Parameters

ParameterTypeDescription
X array-like (n_samples, n_features) Feature matrix of samples to classify.

Returns

np.ndarray of shape (n_samples, n_classes) — binary predictions.

y_pred = market.predict(X_test)

from sklearn.metrics import f1_score, classification_report
print(f1_score(y_test, y_pred, average='macro'))
print(classification_report(y_test, y_pred))

🧬 evolve

market.evolve() → None

Manually triggers one evolutionary rebalancing cycle across all sectors. Usually called automatically every rebalance_period days, but you can call it explicitly if needed. 

# Force an immediate evolution step
market.evolve()

🔧 MultiLabelWrapper

class MultiLabelWrapper(estimator)

Adapter that makes any sklearn estimator whose predict_proba returns a list of arrays (e.g. MultiOutputClassifier) compatible with BemeMarket.

BemeMarket expects predict_proba(X) to return a single (n_samples, n_classes) array. MultiOutputClassifier returns a list of (n_samples, 2) arrays — one per class. This wrapper normalises that into the expected shape.

When is it needed?

EstimatorNeeds wrapper?
MultiOutputClassifier(...)✅ Yes
ClassifierChain(...)✅ Yes
Custom model returning (n, n_classes)❌ No
 
from beme import MultiLabelWrapper
from sklearn.multioutput import MultiOutputClassifier
from sklearn.svm import SVC

# SVC needs probability=True for predict_proba
base = MultiLabelWrapper(
    MultiOutputClassifier(SVC(probability=True, kernel='rbf'))
)
market.initialize_market(base, X_train, y_train)

🎛 Hyperparameter Guide

Parameter Symbol Default Description
n_funds N 50 Number of hedge fund agents. More agents = more diversity but slower per-sample training. 30–100 is a practical range.
leverage L 25.0 Scales P&L magnitude. High values accelerate learning but can bankrupt good agents on noisy samples. Lower to 5–10 for noisy or time-series data.
temperature T 1.5 Controls consensus softness. T → 0: richest agent monopolises the price. T → ∞: equal vote. NLP/text: 1.5–2.5. Tabular: 1.0–1.5.
bounty_map βc 'auto' recommended 'auto' — computed from class frequencies: β_c = 1.0 + log(max / count_c). Frequent classes stay near 1.0; rare classes get a log-proportional boost.
None — default map {1:1.5, 2:2.0, 7:1.5, 8:2.0}.
{} — disables bounty entirely.
rebalance_period H 100 Days between evolutionary cycles. Lower values = more aggressive replacement. Recommended range: 50–200 for datasets with 1000–10000 samples.

Tuning Recipes

NLP / sparse text (TF-IDF, embeddings):
Keep temperature high (1.5–2.5). Logistic models on text tend to produce extreme probabilities (very close to 0 or 1). High temperature softens the weighted average and prevents one agent from dominating the consensus based on a single overconfident prediction.
Noisy / time-series signals:
Keep leverage low (5–10). Random noise spikes can cause large one-off losses that bankrupt successful agents before they've built enough history to be protected by evolution.
Heavy class imbalance:
Assign high bounty_map multipliers (2.0–5.0) to the rarest classes. Without bounty, agents learn to ignore minority classes because correctly predicting the majority is economically optimal.

🏗 How It Works

1 · Sectoral Isolation

Agents are divided into two rooms: COMMODITY (classes 1, 2, 4, 7, 9) and FINANCE (classes 0, 3, 5, 6, 8). Each room covers a disjoint set of classes. A class is priced only by the agents in its sector, which reduces cross-class noise and lets agents develop deep expertise in their assigned label subset rather than spreading capacity thinly across all classes.

2 · Economic Incentive — Bounty

Rare classes carry higher βc multipliers. An agent that correctly detects a minority class earns disproportionately more balance than one that detects a common class. This creates automatic economic pressure: agents that ignore rare classes fall behind in wealth and eventually get replaced, while those that specialise in detecting them accumulate capital and gain more influence over the final consensus price.

3 · Evolutionary Selection

Every H days the weakest agent (lowest balance) in each sector is replaced by a deep copy of the strongest (highest balance). A 10% balance transfer is made from parent to child, giving the clone a fresh financial start (at least 100.0) while slightly penalising the parent to keep competitive pressure alive. Over many cycles this propagates successful strategies across the sector while continually pruning underperformers.

📐 Theory & Math

Market Price Discovery

For each class c, the market price is a temperature-scaled sigmoid of the balance-weighted average logit:

Pc = sigmoid( L̄c / T )

L̄c = Σi ∈ Sector_c( Bi · logit(pi,c) )
───────────────────────────────────────────
Σi ∈ Sector_c( Bi )

where p_{i,c} is clipped to [0.01, 0.99] before logit

Agent P&L

After observing the true label, each agent's balance is updated:

ΔBi = Σc ∈ active_classes_i [ ( ytrue[c] − Pc ) ← market was wrong by this much × ( pi,c − Pc ) ← agent beat the market by this much × riski ← agent's own risk appetite × L ← leverage × βc ← bounty multiplier ]

Bi = max(1.0, Bi + ΔBi)

Auto-Bounty v0.2.0

When bounty_map='auto', multipliers are derived from label frequencies in the training set. Frequent classes stay at 1.0; rare classes get a logarithmic boost proportional to their scarcity.

β_c = 1.0 + log( max(counts) / counts_c )

counts_c = number of training samples where class c = 1
If class c appears in every sample → β_c = 1.0 + log(1) = 1.0 (no boost)
If class c is 10× rarer than the most common → β_c ≈ 1.0 + 2.3 = 3.3

Evolutionary Cloning

child.model = deepcopy(parent.model)
transfer = parent.balance × 0.1
parent.balance -= transfer
child.balance = max(100.0, transfer)

🗺 Architecture

Input x_i | v [Agent 1 .. N] predict_proba(x_i) → p_{i,c} clipped to [0.01, 0.99] | v Market Price Discovery For each class c: L̄_c = balance-weighted logit average (over Sector_c agents only) P_c = sigmoid( L̄_c / T ) | v y_hat[c] = 1 if P_c > 0.5 else 0 ← returned immediately | v (training mode only — true label is known) P&L + Bounty Update ΔB_i = Σ_c [ (y_true[c] − P_c) × (p_{i,c} − P_c) × risk_i × L × β_c ] B_i = max(1.0, B_i + ΔB_i) | v Every H days → Evolutionary Rebalancing COMMODITY sector: clone top → replace bottom FINANCE sector: clone top → replace bottom
